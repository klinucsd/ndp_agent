# WCS Clip: Annual Burn Probability for San Diego County

Example requests sent to DeepNDP for discovering and clipping geospatial web services. All files in this directory were generated by DeepNDP in response to the requests below.

---

## Step 1

### Request

```
Find a WFS layer for California counties in the CKAN catalog at https://wifire-data.sdsc.edu,
using a query limit of 3
```

---

## Step 2

### Request

```
Identify a WCS layer in the same CKAN that provides annual burn probability data
```

---

## Step 3

### Request

```
Using these two layers, write a Python script that clips the WCS annual burn probability data
to the boundary of San Diego County derived from the WFS layer and saves the clipped result
as a GeoTIFF file. Note that the two services use different coordinate reference systems,
with the WCS in EPSG:3310 and the WFS in EPSG:4269.

Because downloading WCS data is time-consuming, do not use OWSLib; instead, issue direct HTTP
requests and restrict the WCS request to the bounding box of San Diego County rather than
downloading the full coverage. The script should be tested and debugged to ensure it runs
correctly and produces the expected GeoTIFF output.

Don't use the http_request tool to fetch data from the WCS and the WFS. Write Python script
to explore them and fetch data from them.

Afterward, convert the working Python script into a Jupyter Notebook that includes appropriate
visualizations of the clipped result. Maps in the visualizations should use the projection
EPSG:4269. Add a notebook cell that installs required dependencies using ! pip install ...
(do not use ! uv pip install inside the notebook), and verify that the notebook executes
successfully in a JupyterHub environment.

A virtual environment created with uv is available in the parent directory and may be used
as needed.
```
